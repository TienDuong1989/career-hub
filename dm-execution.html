On-site Checklist
Run per page — title, H1, keywords, CTA, links
▾
Run this checklist every time you publish or optimize a page. Different from Technical SEO — this is per-page, repeated for every new page.
Title tag includes target keyword · under 60 chars
Meta description compelling · under 155 chars
H1 unique · one per page · keyword included
H2/H3 structured · secondary keywords in headings
Keyword in URL · first 100 words · image alt text
Internal links: to pillar page + 2 related pages
CTA visible above fold · action verb · one goal per page
Word count sufficient for intent (service: 800+, blog: 1500+)
Run per page · not one-timeGoogle Drive ↗
⚙️
Technical SEO
Infrastructure · Core Web Vitals · Crawlability
▾
Infrastructure-level setup. Do once, monitor quarterly. Different from On-site — this applies to the whole website, not individual pages.
Core Web Vitals: LCP <2.5s · CLS <0.1 · FID <100ms
Crawlability: XML Sitemap submitted · Robots.txt correct
301 Redirects map: old URLs → new URLs (no 404s)
Schema markup: Organization + Service + FAQ
HTTPS enforced · Mobile-friendly (Google Mobile Test)
Index coverage: Google Search Console · fix errors
Duplicate content: canonical tags set correctly
Audit quarterly · monitor monthlyGoogle Drive ↗

Content Cluster & Topic Authority
Pillar page → Cluster pages → Internal links
▾
How to organize content so Google recognizes you as an authority on a topic. Structure first, then write — not the other way around.
1 Pillar page — broad topic, e.g. /software-development
Cluster pages — sub-topics linking back to pillar
Rule: every cluster page links to pillar page (always)
Same topic cluster = same URL folder structure (/finance/*)
One keyword per page — avoid cannibalization
Plan structure before writingGoogle Drive ↗

CRO Checklist
CTA · Form · Trust · Mobile
▾
Systematic checklist to run on any page to improve conversion rate. Check before and after every redesign.
CTA: visible above fold · action verb · contrast color
Form: ≤ 5 fields · clear labels · no CAPTCHA on first visit
Trust: logo bar · review count · security badge
Mobile: tap targets ≥ 44px · readable font size ≥ 16px
Load time < 3s · no layout shift on load
